# scene-object-detection
Scene object classification and segmentation project
https://www.youtube.com/watch?v=NZuL1JPh10c 
Цель работы: реализовать графический интерфейс пользователя (GUI) на базе PyQt5, обеспечивающий удобную загрузку изображений, интерактивный просмотр с масштабированием и панорамированием, переключение режимов обработки, работу с вебкамерой в реальном времени, вывод аналитики, экспорт результатов и корректное завершение работы приложения.
Ход работы
1. Создание приложения на PyQt5 с поддержкой загрузки изображений через диалоговое окно
Реализуется главное окно приложения на базе QWidget. В левой части располагается панель управления, в центральной — область для отображения изображения, в нижней — панель для вывода аналитики.
Реализуется кнопка «Загрузить изображение», при нажатии на которую открывается стандартное диалоговое окно выбора файла (QFileDialog). Поддерживаются форматы: JPG, PNG, BMP. Путь к выбранному файлу сохраняется в переменной для последующей обработки. После загрузки имя файла отображается на кнопке для подтверждения пользователю.
2. Реализация интерактивного просмотра изображений: масштабирование колесом мыши (0.2x–6.0x) и панорамирование перетаскиванием (click-and-drag)
Реализуется специальный класс на основе QLabel (ClickableLabel), который обрабатывает события мыши:
•	Масштабирование: При вращении колёсика мыши коэффициент масштаба изменяется (умножение на 1.15 при прокрутке вверх, деление на 1.15 при прокрутке вниз). Коэффициент ограничивается диапазоном от 0.2 до 6.0. После изменения масштаба изображение перерисовывается с новым коэффициентом.
•	Панорамирование: При зажатой левой кнопке мыши и перемещении курсора вычисляется смещение (delta), которое прибавляется к текущему сдвигу изображения. После отпускания кнопки панорамирование прекращается.
•	Перерисовка: Изображение масштабируется с учётом текущего коэффициента, затем смещается на величину сдвига и центрируется в области отображения.
3. Реализация трёх режимов обработки с выводом результата: YOLO / Depth / All
В панели управления размещаются три радиокнопки (QRadioButton) для выбора режима:
•	YOLO: Применяется только детекция объектов с помощью YOLOv8n. На изображение наносятся ограничивающие рамки с подписями классов и уверенностью. Выходное изображение — исходное с нарисованными рамками.
•	Depth: Применяется только модель оценки глубины Depth Anything V2. Строится карта глубины, которая нормализуется и раскрашивается цветовой картой MAGMA. Выходное изображение — тепловая карта глубины.
•	All: Оба режима выполняются последовательно. Результаты детекции и карта глубины объединяются горизонтально (располагаются рядом) в одно выходное изображение.
По умолчанию выбран режим All. При смене режима и нажатии кнопки «Запустить» выполняется обработка с соответствующими параметрами.
4. Добавление вебкамеры для анализа в реальном времени с выбором режима (YOLO для детекции, Depth для глубины с отображением расстояния до центра кадра)
Реализуется кнопка «Webcam режим», при нажатии на которую открывается диалог выбора режима работы с камерой. Пользователь выбирает один из двух вариантов:
•	YOLO: Захват кадров с вебкамеры через cv2.VideoCapture(0). Каждый кадр передаётся в модель YOLO для детекции объектов. Результат (изображение с нарисованными рамками) отображается в отдельном окне OpenCV в реальном времени. Выход из режима — по нажатию клавиши ESC.
•	Depth: Захват кадров с вебкамеры. Для снижения нагрузки модель глубины применяется не на каждом кадре, а на каждом третьем. В центре кадра выделяется прямоугольная область, вычисляется медианная глубина и пересчитывается в метры по формуле (0.5 + ((255 - median) / 255) × 9.5). Расстояние отображается текстом на экране. Выход из режима — по нажатию клавиши ESC.
Для Depth режима создаётся отдельный поток (DepthThread), который обрабатывает кадры асинхронно, чтобы не блокировать основной поток приложения.
5. Реализация группы аналитики в нижней панели: группировка объектов, средняя уверенность, средняя дистанция, логический вывод сцены
В нижней части главного окна располагается панель с текстовым полем (QTextEdit) для вывода аналитики. После обработки изображения формируется отчёт, содержащий:
•	Группировка объектов: Обнаруженные объекты распределяются по категориям (Люди, Транспорт, Животные, Мебель, Техника, Интерьер, Предметы, Прочее). Для каждой категории выводится количество объектов, средняя уверенность детекции, средняя дистанция (если доступна) и детальный состав по классам.
•	Логический вывод сцены: На основе комбинации детекций YOLO и классификации ViT формируется текстовый вывод: обнаружена ли дорожная сцена, интерьер помещения, сцена на природе, а также подсчитывается количество людей.
•	Общая статистика: Общее количество обнаруженных объектов.
Результат аналитики выводится в текстовое поле, доступное для чтения и копирования.
6. Экспорт результатов обработки в файл (PNG/JPG)
Реализуется кнопка «Экспорт», при нажатии на которую открывается диалог сохранения файла (QFileDialog.getSaveFileName). Пользователю предлагается выбрать формат (PNG или JPG) и указать имя файла. Текущее выходное изображение (результат обработки) сохраняется по указанному пути с помощью cv2.imwrite(). После успешного сохранения выводится информационное сообщение.
7. Корректное завершение работы: освобождение памяти моделей, закрытие потоков вебкамеры, выход из приложения
Реализуется корректное завершение работы приложения при закрытии главного окна или при выборе пункта меню «Выход»:
•	Останавливаются и завершаются все активные потоки (в частности, поток вебкамеры DepthThread, если он был запущен);
•	Закрывается соединение с вебкамерой (cap.release()) при активном режиме;
•	Закрываются все OpenCV-окна (cv2.destroyAllWindows());
•	Модели остаются в памяти до завершения процесса — специального освобождения не требуется, так как память освобождается автоматически при выходе из приложения;
•	Завершается главный цикл QApplication.
В результате было реализовано приложение на PyQt5 с загрузкой изображений, масштабированием и панорамированием, тремя режимами обработки (YOLO/Depth/All), вебкамерой в реальном времени, выводом аналитики и экспортом результатов. Корректное завершение работы обеспечено.
